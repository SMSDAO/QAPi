QAPi Documentation

QAPi is a global, virtualized module provider — a "Streaming Service for Code." Instead of installing dependencies locally, your project resolves them through the QAPi SDK, which dynamically links to globally hosted repositories on GitHub or private VPS instances.

⚡ Starter

Free. Public GitHub modules. 100 calls/min.

🚀 Pro — $4/mo

Private VPS modules. Dedicated cache. 1 000 calls/min.

🔐 Audited — Custom

Zero-day scanning. 99.9% SLA. Unlimited calls.

Quick Start

1. Get an API key

Sign up at qapi.dev/signup or call the API directly:

curl -X POST https://api.qapi.dev/auth/signup \
  -H "Content-Type: application/json" \
  -d '{"email":"you@example.com","tier":"starter"}'

2. Install the SDK

npm install @qapi/sdk

3. Resolve your first module

const { QAPiClient } = require('@qapi/sdk');

const client = new QAPiClient({ apiKey: 'qapi-starter-YOUR_KEY' });

const result = await client.resolve('express');
console.log(result.entrypoint); // → https://github.com/expressjs/express…

Authentication

All protected endpoints require your API key. Supply it in one of two ways:

# Option 1: Custom header (recommended)
X-QAPi-Key: qapi-starter-YOUR_KEY

# Option 2: Bearer token
Authorization: Bearer qapi-starter-YOUR_KEY

Keys are prefixed with their tier: qapi-starter-, qapi-pro-, qapi-audited-.

SDK – Installation

npm install @qapi/sdk
# or
yarn add @qapi/sdk

The SDK has zero dependencies. It uses Node.js built-in http/https modules.

QAPiClient

const { QAPiClient } = require('@qapi/sdk');

const client = new QAPiClient({
  apiKey:  'qapi-starter-YOUR_KEY',   // required
  baseUrl: 'https://api.qapi.dev',    // optional, default shown
  timeout: 10_000,                    // optional, ms
});

client.resolve(name, version?)

Resolves a module node and returns its entrypoint URL. Results are cached in-process by name@version.

const result = await client.resolve('express');
// or pin a version:
const result = await client.resolve('express', '4.18.2');

// Returns:
{
  resolved: true,
  name: 'express',
  version: '4.18.2',
  entrypoint: 'https://github.com/expressjs/express.git#master:index.js',
  sourceType: 'github',
  audit: { score: 98, passed: true, zeroDay: false },
  cachedTtlSeconds: 3600,
}

Throws QAPiError with statusCode 403 (tier too low) or 404 (not found).

client.list(filter?)

const { count, tier, modules } = await client.list({ name: 'express' });

client.audit(name, version?) [Audited tier]

Triggers a real-time security scan. Requires the Audited tier.

const result = await client.audit('express');
// Returns:
{
  name: 'express', version: '4.18.2', id: '...',
  audit: { score: 98, passed: true, vulnerabilities: { critical: 0, ... }, zeroDay: false },
  status: 'active',
}

API Reference

GET/healthNo auth required

{ "status": "ok", "service": "qapi-core", "timestamp": "2026-03-11T05:00:00Z" }

GET/auth/tiersNo auth required

Returns a description of all available subscription tiers.

POST/auth/signupNo auth required

Creates a new API key.

// Body
{ "email": "you@example.com", "tier": "starter" }

// Response 201
{
  "message": "API key created successfully.",
  "apiKey": "qapi-starter-a1b2c3d4-...",
  "tier": "starter",
  "email": "you@example.com",
  "tierConfig": { "price": "Free", "callsPerMin": 100, "features": [...] }
}

GET/modulesRequires API key

Lists all module nodes accessible to your tier. Optional query: ?name=<search>.

GET/modules/resolve?name=&version=Requires API key

Resolves a module by name (and optionally version) and returns its entrypoint URL.

GET/audit/scan?name=Audited tier

Triggers a real-time security scan on a module node. Returns audit scores, CVE list, and zero-day flag.

GET/audit/reportAudited tier

Returns a full audit summary across all module nodes.

Module Node Schema

Every module in the QAPi registry is described by a Module Node JSON object. The full JSON Schema is available at module-node.schema.json in the repository root.

{
  "id":      "a1b2c3d4-e5f6-4789-ab01-23456789cdef",  // UUID v4
  "name":    "express",
  "version": "4.18.2",
  "source": {
    "type":       "github",          // github | vps | npm | private
    "url":        "https://github.com/expressjs/express.git",
    "branch":     "master",
    "entrypoint": "index.js",
    "privateVps": false,
    "region":     "us-east-1"
  },
  "tier":   "starter",               // starter | pro | audited
  "status": "active",                // active | degraded | offline | deprecated | quarantined
  "audit": {
    "score":           98,           // 0–100
    "passed":          true,
    "vulnerabilities": { "critical":0, "high":0, "moderate":0, "low":1, "info":2 },
    "cve":             [],
    "zeroDay":         false,
    "lastScannedAt":   "2026-03-11T05:00:00Z",
    "scanEngine":      "qapi-audit-v1"
  },
  "cache":      { "enabled":true, "ttlSeconds":3600, "dedicatedCache":false, ... },
  "registry":   { "npm":"express", "github":"expressjs/express", "license":"MIT", ... },
  "metrics":    { "callsTotal":5000000, "avgLatencyMs":4.2, ... },
  "timestamps": { "createdAt":"...", "updatedAt":"...", "lastResolvedAt":"..." }
}

bootstrap.ps1 — Virtual Node Orchestrator

The bootstrap.ps1 script configures your local machine to use QAPi as the module resolver instead of node_modules.

# Windows / PowerShell 7+
irm https://raw.githubusercontent.com/SMSDAO/QAPi/main/bootstrap.ps1 | iex

# Then in your project:
$env:QAPI_KEY = "qapi-starter-YOUR_KEY"
node -r @qapi/sdk/register ./your-app.js
